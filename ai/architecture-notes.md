# Architecture Notes

Last updated: 2026-07-17

## Current Architecture

- The project is a Unity 2D/URP project with a thin runtime-generated MVP strategy layer.
- `StrategyGameBootstrap` installs scene load/unload hooks, gates gameplay through `StrategySceneCatalog`, and creates one scene-local `StrategyGameContext`. The context owns typed service registration, exact-scene lookup, lifecycle state (`Created`/`Configuring`/`Ready`/`Failed`/`Disposed`), and the bootstrap pause owner. `StrategyBootstrapRunner` flattens nested enumerators, reports failures to the context, and releases lifecycle state on cancellation/unload; this supports menu-to-game and game-to-menu transitions without stale static scene services.
- `StrategyMainMenuBootstrap` owns the `MainMenu` scene entry point. `StrategyMainMenuController` builds the full-screen Continue/New/Settings/Quit UI, starts only the HUD SFX mix, and removes any stale music controller; `StrategyMainMenuButtonHover` animates pointer/selection movement, tint, label, gold accent, press state, and one throttled low-volume hover cue. `StrategyMainMenuBackdrop` cover-crops one dedicated generated point-filtered static pixel-art image without runtime backdrop animation or gameplay sprite-factory dependencies. `StrategyGameSettings` persists master/music/effects volume plus fullscreen state.
- `StrategyPauseMenuController` is the gameplay-owned Escape menu created by bootstrap after persistence. Its code-built view dims the live map behind a Labyrinth-inspired dark left panel with gold accents; its actions expose Resume, `StrategySaveSystem.SaveNow`, shared `StrategyGameSettings`, and confirmed Main Menu/Quit transitions. While open it owns an `All`/`Close` input context and a named `StrategyTimeScaleController` pause lock, so map input is blocked and the requested x1/x2/x3 speed survives Resume. Cancel consumption is frame-guarded in `StrategyInputRouter`, preventing the Escape press that closes another top HUD from reopening the pause menu in `LateUpdate`.
- `StrategyFoundingJourneyBootstrap` and the partial `StrategyFoundingJourneyController` own the New Settlement interstitial. They build four Resources-backed pixel-art story panels, stable-ID question cards, summary/loading states, reduced-motion handling, centralized input context, bounded map-wait/error recovery, and the final decision handoff without creating a gameplay context. `StrategyFoundingJourneyPresentation` owns cover-cropped authored shot motion, shot/atmosphere crossfades, opening/staged reveals, and cinematic chrome; `StrategyFoundingJourneyAtmosphere` uses a normalized reference frame under the active artwork transform; `StrategyFoundingJourneyAudio` asynchronously loads scene-local wind/rain/fire loops and crossfades them through the existing mixer buses.
- `StrategyStartSiteMapSnapshot` captures only map data needed by the pure deterministic `StrategyStartSiteSelector`. The selector keeps safety/playability as hard constraints, ranks valid cells from water/landscape/livelihood/priority preferences, reserves the exact `3x3` Caravan Cart blocker, uses stable hash tie-breaking instead of `UnityEngine.Random`, and exposes explicit relaxed/legacy failure diagnostics. `StrategyFoundingStartState` travels on the prepared map and bridges that result into bootstrap and persistence.
- `StrategyMapPreloadCoordinator` survives menu-to-founding-to-game scene changes, validates the existing save before presenting Continue, chooses one likely Continue/New seed, prewarms starter visuals plus short HUD/footstep clips, incrementally generates deterministic map cells, paints the terrain `Color32` buffer through a cancellable parallel worker, and keeps only one prepared map candidate in memory. Continue opens gameplay directly; New Settlement opens the founding journey immediately while preparation continues, then gameplay explicitly transfers the same candidate into its context. Long Music/Nature folders remain owner-loaded in gameplay; orphan candidates are disposed instead of relying on implicit static ownership.
- `StrategyDebugLogger` is created before other strategy systems, writes structured session lines to project-root `debug.log` in the Unity Editor and `<game directory>/Logs/debug.log` in standalone builds, mirrors Unity log messages, and exposes static `Info`/`Warn`/`Error` helpers. If the portable build directory is not writable, it switches its canonical path to `Application.persistentDataPath/debug.log`, including during later recovery. `StrategyLogFileRotation` preserves the previous session, rotates at 8 MiB, retains the newest three archives under the matching `Logs/StrategyDebug` tree, and uses a UTC/PID live fallback when another process locks the canonical file.
- `StrategyPerformanceDiagnostics` records allocation-free per-frame samples into 10-second stable-context windows split by population band, time phase, weather, and requested simulation speed. It resets around application focus/pause transitions so Editor Alt-Tab time is not reported as a gameplay hitch, logs frame percentiles, memory/GC, active world counts, and resident path/decision counters; `-strategyBenchmarkSeed` can force deterministic map generation for repeat sessions. Fixed `ProfilerMarker`s cover navigation orchestration/A*, resident task selection, AutoWorkforce ticks, cinematic-light scan/LOD refresh, and terrain painting without dynamic marker allocation.
- `StrategyRuntimeObjectCreationGuard` blocks loose-resource pile factories from creating new scene GameObjects while Play Mode is shutting down or when runtime code is not actually playing.
- `CityMapController` owns 192x192 default map-cell data, deterministic seed-driven terrain, minimum Forest coverage, River/Lake identity, relief, walkability versions, and the generated point-filtered terrain texture. Normal menu and direct-gameplay bootstrap both use staged cancellable generation with worker-painted pixels and main-thread texture upload; synchronous generation remains an explicit regeneration/test fallback. Prepared-map ownership and presentation are transferred explicitly.
- `StrategyWorldChunkRegistry` is created after starter placement and divides the generated map into 16x16 cell chunks. It indexes placed buildings, active construction sites, and residents, exposes camera-near and active-settlement chunk flags, and provides dirty flags plus active/all chunk query APIs, read-only active building/site views, and cached building-component queries; Storage Yard, Granary, household-food fallback, construction-site dispatch, and auto-workforce worksite reads route through it with global fallback, and fog/weather visual repaint paths use it to limit texture uploads to active chunks after initial full paints.
- `StrategyTerrainTexturePainter` paints the generated map into one managed `Color32` buffer using main-thread-prewarmed catalog swatches for Grass, Meadow, Forest, Dirt, Shore, and Water. Each tile builds one cached paint context/catalog sample set outside the inner pixel loop; the parallel worker adds smooth macro variation, neighbor transitions, and relief without reading Unity objects. Map shaping classifies kind/water before relief and reuses that water mask in the second pass; the default grid stays hidden and `CityMapController` performs one texture upload.
- `StrategyVisualCatalog` is the catalog-first migration seam from runtime-drawn sprites to authored art. The Editor baker outputs editable PNGs for buildings, resident poses/portraits, nature, terrain, construction, roads, work cycles, and stock levels; matching assets under `Visual/Authored` survive baseline deletion, pass explicit manifest/import/layout validation, and replace rebuilt catalog entries. Authored coverage includes the existing 17-tool Build catalog plus the Starter Caravan Cart: 47 static non-Bridge final sprites, 46 matching seven-stage construction sequences, one shared six-frame Chicken Coop animation, and 12 modular Bridge sequences. The newer Scout Lodge intentionally uses dedicated procedural final/construction fallbacks until an authored family is added. Normal families preserve procedural world dimensions through higher-density PPU contracts, while `StrategyBridgeVisualProfile` composes centered `48 PPU` horizontal/vertical Start, Middle, and End modules for spans `3-12`. Runtime factories still query typed catalog entries before procedural fallback.
- `StrategyBuildingVisualAlignment` owns final/construction pivots, `StrategyBuildingVariantProfile` owns single-variant normalization, and `StrategyBuildingVisualAnchorProfile` owns stock, work, effect, and light anchors coupled to authored geometry. `StrategyChickenCoopVisualProfile` adapts one authored animation to standalone and legacy House-upgrade scales, while building snow extraction supports normal non-readable imports through GPU readback. Carried resources and live stock/work overlays remain separate runtime layers rather than being baked into base sprites.
- Seasonal visual work is centralized: one fixed spring/autumn detail pool follows the camera, one low-frequency nature-controller pass tints vegetation, and existing seasonal surfaces own snow/ice/building caps. Camera event feedback is one camera component and never adds Update work to event sources.
- `StrategyUiThemeProvider` supplies one Resources-backed Inter font to runtime-created text and bounded post-scene scans, while retaining the existing sliced pixel panel/button frames; typography ownership stays centralized across the main menu, founding journey, HUD, dialogs, and debug UI.
- `StrategyUiButtonFeedback` provides shared unscaled pointer/focus/press response and separately throttled hover audio for code-built player buttons without replacing semantic action sounds. `StrategyUiPanelTransition` provides interruptible unscaled fade/slide/scale, reduced-motion fallback, immediate control disable, and close-time raycast shielding for runtime HUD panels and dialogs.
- `StrategyInGameCinematicPlayer` is the reusable owner for gameplay-space microcinematics. It captures the exact strategy-camera view, owns one all-channel/swallow input context, a full-screen Unity-UI raycast shield with temporary selection clearing, and a named time-scale pause lock. It invokes each sequence's pre-roll `Begin` hook before the camera moves, blends focus and 2.39:1 letterbox bars together with unscaled reduced-motion-aware timing, and retains ownership/reentry exclusion through a non-modal camera return. Sequence cleanup is idempotent, while the resolved callback runs before restoration so a modal story can synchronously claim input, pause, and UI selection without an exposed gameplay frame or being overwritten by cleanup.
- `StrategyNavigationService` is the shared runtime path owner for residents and ordinary land agents. It uses reusable array-backed A*, preserves resident diagonal/corner/trail-cost smoothing and wildlife River transit rules, coalesces duplicate deferred requests under an 8-path/2.2 ms frame budget, and drains critical then normal then background queues so wildlife roaming cannot starve resident work. Deferred resident decisions stop scanning and retry without recording path failure; wildlife candidate scans also yield on deferral. Short-lived path buffers use pooling plus bounded recency eviction instead of clearing the entire cache, and all results invalidate when `CityMapController.WalkabilityVersion` changes.
- Resident behavior is split behind `StrategyResidentMovement`, `StrategyResidentInventory`, `StrategyResidentTaskState`, `StrategyResidentTaskExecution`, and focused household-cooking/task-arrival helpers; `StrategyResidentAgent` remains the Unity-facing coordinator and compatibility surface for existing work/home/funeral flows. Planned task kind is authoritative in the execution table, and characterization tests lock the current priority/order contract.
- Scout exploration follows the existing worksite/resident split: `StrategyScoutLodge` owns its one-worker roster, strict free-worker automation eligibility, explicit appointment/reassignment eligibility, frontier reservations, temporary unreachable-target rejection, and HUD status; `StrategyResidentAgent.TryReleaseExternalWorkAssignment` removes an explicitly selected transfer through the previous owner APIs and preserves any required carried-resource return. Pure `StrategyScoutTargetSelector` chooses the nearest walkable unknown-side frontier cell, while `StrategyResidentAgent.Scouting` owns critical-priority navigation, short surveys, point-of-interest priority, continuous day/night duty, and cleanup. `StrategyScoutLodgeOnboardingController` coordinates the first live Lodge's temporary unscaled camera focus with captured-view restoration, pause/input handoff, and exact-resident dialog. Fog state is not mutated directly because the moving Scout remains an ordinary resident reveal source.
- `StrategyPointOfInterestController` is the map-owned landmark and Coal/Iron site owner. Pure `StrategyPointOfInterestPlacement.SelectMineralPlans` selects the nearest introductory neutral landmark plus nine separated camp-connected resource landmarks; typed landmarks alternate Coal/Iron with a deterministic five/four split, own distinct `2x2` mineral footprints 3-5 cells away, reserve a usable `2x3` extraction block, and keep every mineral more than 24 cells from the camp and more than 6 cells from the neutral point. `StrategyNaturePropController` materializes only those planned deposits through an explicit POI API; it no longer scatters or guarantees global/starter Coal or Iron. Each point persists its role, exact mineral origin, remaining amount, and investigated state. Clearing or rolling back the registry removes its live deposits synchronously, and restore falls back atomically to a fresh deterministic layout instead of retaining a partial set. Persistent Fog of War still gates targeting; investigation queues a resource-specific one-button report under the existing modal pause/input contract.
- `StrategyResourceStore` is the common physical-stock owner for storage, production buildings, houses, the starter cart, loose construction piles, and loose carried-resource piles. `StrategyResourceQueryService` aggregates stock by scope and reads exact loose prepared-dish ration payloads instead of treating every transported dish bundle as fallback food, while reservation-provider adapters preserve existing source-specific reservation rules without duplicating availability arithmetic in HUD, construction, and seasonal readiness code.
- `StrategyFirstWinterController` extends starter onboarding into a food/fuel preparation phase and a winter endurance goal. House warmth, resident cold exposure, seasonal refugee pressure, and shortage consequences form the current progression slice; after the first winter it clears the goals and continues as a sandbox, with no victory or defeat state.
- `StrategyFirstNightFaunaEventController` coordinates the Day 1 `Dusk`/`Night` thresholds, guarantees the three pending mice in one refresh, and runs `StrategyFirstNightRatCinematic` before the three-frame fullscreen chronicle. Its deterministic shot planner chooses a nearby adult and a walkable 4-6-cell corridor, with reversible staging for hidden/sleeping fallbacks; the transient actor reuses the exact standard settlement mouse sprite/scale with transform-only motion, while reusable pulsing world rings mark both participants and the resident's dedicated `MouseStartle` visual override restores exactly. `StrategyFirstNightFaunaStoryController` then owns the modal story lifecycle and reuses the Founding Journey presentation/atmosphere path. The story callback remains the sole unlock/create path for the first world cat. Fauna agents remain runtime-only, and the existing version-7 `Dormant`/`MiceVisible`/`StoryCompleted` stage treats `MiceVisible` as the entire unresolved rat-plus-story presentation after load.
- `StrategySaveSystem` owns version-7 JSON persistence, v1/v2/v3/v4/v5/v6 migration, structural/domain validation, a 32 MiB file guard, bounded top-level and nested resident-child/prepared-dish/point-of-interest collections, atomic temporary-file replacement, and `.bak` recovery. Snapshots use stable string IDs rather than Unity object references and restore the founding profile/answers/exact camp/current cart origin plus map seed/time/weather, first-winter milestones, first-night fauna stage, buildings and construction, resource/dish stores, residents and kinship, loose resources including exact prepared-dish payloads, explored fog, route-road cells, and point-of-interest geometry/resource role/mineral origin/remaining amount/investigated state. Pending positive POI sites reserve their marker and full extraction block before deterministic nature/forage generation, then release those temporary counters immediately before restore; depleted sites retain their typed landmark without recreating a deposit. The v5-to-v6 migration regenerates the owned POI layout around the restored world; v6-to-v7 derives the first-night fauna stage from saved calendar time so established settlements do not replay the chronicle. Save capture flushes accepted deferred demolition and records in-transit resident stock as loose resources at the resident's saved cell.
- `StrategyVerificationRunner` provides deterministic lifecycle/input/save/audio/map contracts, including explicit map-seed behavior and procedural plus production-16px catalog terrain golden hashes. Play Mode checks cover menu isolation, the complete prepared menu-to-founding-to-gameplay launch with exact camp/cart/forage invariants, and direct gameplay bootstrap under wall-clock progress/stall watchdogs and unexpected-error collection with a narrow batch-only allowlist. The fixed-seed two-day soak lets the first-night rat cinematic finish on unscaled time, asserts its transient actor/mouse/cat and modal-ownership invariants, advances all three chronicle frames, then continues exercising refugee and point-of-interest modals under both final and peak 128 MiB memory-growth budgets. NUnit EditMode tests live in a dedicated test assembly; 1600x900 menu and Noon/Spring/Autumn/Night/Winter gameplay captures still require a real graphics device because URP camera rendering is unsupported on Unity's Null graphics backend.
- CI soak verification is tiered. Pull requests and `main` run a 45-game-second `QuickSoak` covering 3 in-game hours and write `Logs/QuickSoakSmoke.txt`; the deterministic 720-game-second full soak covers 2 in-game days and writes `Logs/SoakSmoke.txt` only for the nightly 01:23 UTC schedule, manual `workflow_dispatch`, `release`/`release/**` branches, and `v*` tags. Fast PR/main concurrency cancels stale superseded runs, while full-soak concurrency does not cancel in-progress evidence.
- `StrategyWaterAnimationController` renders one generated point-filtered overlay texture above the static map for animated water depth tint, directional river streaks, lake sparkles, broken shoreline foam, wet shore-edge pixels, and rain/storm ripple highlights, while repainting only cached water/shore cells after initial setup.
- `StrategyTrailController` treats repeated completed resident travel between two different non-Bridge buildings as the road source and requires three traversals of a building pair before committing permanent road cells. Route recording follows accepted source-route cells directly, adds a single-sided connector only for a genuinely rejected/disconnected candidate, uses cardinal repair with a high 2x2-square cost, and verifies endpoint connectivity only while the branch has not already joined the existing network. Full connectivity rebuilds may exceed the source route by only the small local repair allowance. Road eligibility centrally rejects non-walkable nature/Stone/grave cells, non-buildable underground resource/POI/campfire cells, and occupied forage cells; ordinary resident navigation remains free to cross walkable underground fields. It does not proactively step toward nearby existing roads after each accepted cell, avoiding connector-and-repair rectangles around shared destinations. If no bounded square-avoiding repair exists, the original actually traversed cardinal route is committed as the connectivity fallback; route commit no longer independently rejects a required connector. Each building pair keeps a still-valid canonical route so repeated traversals reinforce the same path instead of widening roads with nearby variants. A raw route-membership query protects later static spawns even before an invalid route key is pruned. On Continue, `StrategySaveSystem` provides only validated, dimension-matching defensive trail-cell copies as non-functional bootstrap reservations before nature/forage generation; final save application clears those reservations and revalidates actual road restoration. `StrategyRoadsidePropController` derives sparse non-blocking roadside torch props from eligible straight route-road cells and refreshes them when roads or adjacent buildability change.
- `StrategyTrailSpriteFactory` generates connected procedural pixel-art trail sprites for cardinal N/E/S/W right-angle neighbor masks, 3 wear levels, and deterministic variants.
- `StrategyMapDistributionUtility` provides seeded shuffled full-map cell iteration and macro cluster scoring for capped map-object generation.
- `StrategyNaturePropController` is created after map generation and guarantees global Stone/Clay minimums before filling the remaining shared 3600-prop budget with seeded Stone, Clay, and vegetation. Gameplay bootstrap runs those guarantees and the shuffled whole-map pass incrementally under a bootstrap pause; Coal/Iron are excluded from this pass and are created only for POI-owned sites outside the decorative budget.
- `StrategyNaturePropController` supports the camp cell-radius exclusion plus an exact rectangular founding-cart exclusion; `StrategyForageResourceController` consumes the same cart rectangle so neither generated layer overlaps the reserved starter blocker.
- `StrategyNaturePropController` also places generated Stone deposits before vegetation on a cell: standalone Boulders, Rock Clusters, and larger Cliffs that register with `StrategyStoneResourceController`, block their occupied footprints, and use macro cluster weighting instead of purely per-cell random scatter.
- `StrategyPointOfInterestController` asks `StrategyNaturePropController` to create fixed `2x2` Iron Vein or Coal Seam fields only at owned typed landmarks. They register with their existing resource controllers, remain walkable, block normal building placement through the map overlay, and can be mined by matching Mine or Coal Pit worksites.
- `StrategyNaturePropController` places generated near-water Clay fields as multi-cell wet clay patches/banks on walkable shore/land cells within a short water radius; they register with `StrategyClayResourceController`, remain walkable, block normal building placement, avoid adjacent Iron/Coal/Clay fields, use macro cluster weighting, and can be mined by Clay Pit worksites built over them.
- When the startup campfire exclusion center is known, `StrategyNaturePropController` guarantees at least 5 starter-area Stone deposits within `StrategyStonecutterCamp.WorkRadius`; no Coal or Iron is generated in the starter area.
- `StrategyForageResourceController`, `StrategyForageNode`, and `StrategyForageSpriteFactory` create non-blocking Berries/Roots/Mushrooms forage nodes on suitable walkable/buildable non-road land after nature generation through the shared shuffled whole-map pass plus forage cluster weighting, guarantee a small starter forage ring, support resident reservation, despawn gathered nodes immediately after interaction, and queue timed replacement spawns near mature standing trees inside active Forager Camp work radii. All creation paths share a success-returning guard, and existing nodes participate in road eligibility without becoming navigation blockers. It keeps headroom above the initial generated node target, periodically supports under-supplied Forager Camps, applies local cell-density plus per-camp soft caps so forage does not over-cluster, and reads `StrategySeasonCalendar` gameplay profiles so Summer/Spring are more abundant, Autumn favors Roots/Mushrooms, and Winter sharply slows new forage without deleting existing nodes.
- `StrategyWildlifeController` runs after starter placement and fog/placement setup, builds lake fish regions and a river route from generated `CityMapWaterKind` cells, and spawns deer herds, rabbit groups, wolf packs, capped lake fish shoals, one-way pass-through river fish, and ambient birds only in currently hidden daylight-range fog cells within a broad ring around completed buildings or active construction sites; the startup camp is a fallback anchor only when no building/construction anchor exists. `StrategyWildlifeController.Part11.cs` owns the shared hidden-near-settlement candidate checks used by initial spawn, lake/river fish spawn, reproduction birth cells, and migration targets, while migration diagnostics keep a short cooldown for failed target cells so groups avoid repeated abort loops. Land wildlife travel avoids placed buildings, active construction sites, and the campfire within a 4-cell structure buffer while still allowing generated River crossings as transit-only path cells. Wolf pack placement prefers alternating river sides when a generated river route exists, wolf roam can recover from stale pack centers by falling back around the current wolf position and retargeting the pack center, and wolf prey lookup only reserves rabbit/deer surplus above high control thresholds after subtracting active predator reservations and hunter-reserved rabbits. `StrategyWildlifeRiverCrossing` separates land target cells from land-or-River travel cells: deer, rabbits, and wolves can path through generated River cells with slowed swimming movement and ripple visuals, but final path targets and hunter/wolf prey reservations remain on land cells without changing global map walkability or allowing Lake crossings. `StrategyDeerAgent` owns deer sex/life stage, fawn growth, predator reservation, state transitions, local pathing, resident/noisy-work threat checks, swimming crossings, threat-reaction throttling, and frame-based idle/walk/graze/alert/flee/rest animation; `StrategyRabbitAgent` owns the matching rabbit behavior set plus hunter reservation, predator reservation, arrow-hit, missed-arrow flee reaction, death, carcass, butchering-ready states, swimming crossings, and threat-reaction throttling; `StrategyWolfAgent` owns pack roaming, settlement-pressure avoidance, threshold-limited rabbit/deer hunting with a short fast pounce phase, swimming crossings, and frame-based idle/walk/run/stalk/attack/eat/howl animation; `StrategyFishAgent` owns lake-vs-river habitat state, water-cell pathing, river-route following/despawn, fry growth, swimming, feeding, turning, dart/flee behavior, fishing reservation, hooked/caught states, and surface ripple visuals; `StrategyBirdAgent` owns decorative bird state, habitat landing choices, flying/fleeing, pecking, hopping, duck swimming, and flight shadows.
- Land wildlife migration target selection validates both land-region connectivity and an immediately viable first step before committing a new target; failed target cooldowns cover a small nearby-cell radius so repeated aborts do not retarget the same blocked pocket.
- Wildlife renderers are refreshed against Fog of War current `IsCellVisible` cells, so animals remain simulated in explored fog but bodies, readability overlays, shadows, and ripples render only in cells visible right now.
- Wildlife service load is spread under time acceleration: migration categories are processed over successive frames on scaled simulation cadence with one pending pass at most, land groups/packs retain a cardinal migration route and rebuild it only after walkability revision or route mismatch, connectivity checks reuse scratch collections, and no wildlife decision advances while simulation time is paused. Diagnostic throttles may still use real time.
- `StrategyWildlifeController.Part13.cs` owns Hunter Camp-local support spawns: under-supplied camps periodically seed adult rabbits in a hidden 7-11 cell ring inside `StrategyHunterCamp.WorkRadius`, while upgraded camps can seed rare adult deer under stricter local caps. Support spawns reuse normal hidden wildlife candidate checks, structure buffers, global population caps, and local-density guards.
- `StrategyStoneResourceController` / `StrategyStoneDeposit` track runtime Stone resource nodes with deposit kind, footprint, remaining Stone amount, and reservation hooks for future worker jobs.
- `StrategyIronResourceController` / `StrategyIronDeposit` track runtime underground Iron indicators with deposit kind, footprint, remaining Iron amount, reservation state, and mining hooks for `StrategyMine`.
- `StrategyCoalResourceController` / `StrategyCoalDeposit` track runtime underground Coal indicators with deposit kind, footprint, remaining Coal amount, reservation state, and mining hooks for `StrategyCoalPit`.
- `StrategyClayResourceController` / `StrategyClayDeposit` track runtime near-water Clay fields with deposit kind, footprint, remaining Clay amount, reservation state, inspect data, buildability blocking, and mining hooks for `StrategyClayPit`.
- `StrategyForestryController` / `StrategyForestryTree` register generated standalone trees as mature forestry targets, block tree cells while trees exist, support hit-driven chopping, animate tree shake/cut marks/falling, keep fallen trunks as bucking targets, split them into pickup Logs, release walkability after Logs are taken, and plant saplings that grow through 3 visual stages only on buildable non-road, non-POI cells. Small trees yield 3 Logs, while large generated trees and planted mature trees yield 6 Logs. Lumber target queries enforce the requesting camp's work radius and remaining local stock capacity before reservation so a nearer oversized yield cannot mask a usable smaller target.
- `StrategyWindController` creates/configures a Unity directional `WindZone` as the runtime wind source, accepts weather influence boosts, and `StrategyWindSway` adapts the wind values into 2D sprite sway for generated trees, forest groups, and bushes.
- `StrategyWeatherController` owns the runtime weather state, randomly transitions between Clear, Cloudy, LightRain, HeavyRain, Fog, Storm, Snow, and Blizzard, applies season-biased transition weights from the shared calendar, exposes smooth rain/snow/cloud/fog/storm/wind/wetness intensities, boosts the strategy wind source, feeds rain/wind ambience, and lets water animation intensify ripple hits during rain while snowfall remains visual/wind-only.
- `StrategyTemperatureModel` calculates outdoor Celsius temperature from season, time of day, stable daily variation, and active weather/snow/blizzard intensity; it is read by the top HUD and by house warmth state.
- `StrategySeasonalSurfaceController` reads the calendar, outdoor temperature, and weather intensity to paint gradual coherent world-space snow patches over non-water terrain, ice over water cells, and cached snow caps over placed building sprites. Its frozen-water gameplay threshold is the shared source for blocking Fisher Hut fish reservations, cancelling active casts, reporting no catchable fish, and freezing fish movement/animation until thaw.
- `StrategyWeatherVisualController` renders procedural wet-ground/cold-wash, low-density cloud-shadow, low-density heavy-precipitation mist, capped camera-area rain, and capped camera-area snow overlays in dedicated sorting bands around day/night, preview, and fog-of-war layers; cloud-shadow and mist textures repaint active world chunks after their first full frame or inactive full-clear transition, while dense Fog weather rendering is masked through fog-of-war instead of covering visible cells with a uniform mist overlay.
- `StrategyAudioMixController` is the persistent central runtime mix layer. It routes sources into editor-generated Unity AudioMixer groups, owns named bus multipliers for music, ambience, weather, water, settlement, work, footsteps, wildlife, fire, important events, and HUD, blends pause/night/weather/zoom profiles, and evaluates camera-focus attenuation, low-pass, and reverb for world audio. Its application-focus partial defensively enables background updates and uses one combined focus/application-pause gate to save, zero, and restore `AudioListener.volume`; this also mutes HUD sources that intentionally ignore listener pause without changing user settings or simulation time. Founding Journey reuses Weather/Fire routing through a scene-owned non-spatial loop controller, while shared button hover reuses the HUD source with its own cooldown.
- `StrategyAudioVoicePool` owns 18 reusable spatial world voices with per-family cooldown/concurrency limits and importance-aware stealing. Resident footstep/work trigger components and forestry no longer create per-agent/per-system AudioSources or filters; performance diagnostics report active/capacity/dropped voice counts.
- `StrategyWorldAudioDirector` blends fixed-cost procedural day/night settlement, campfire, and winter layers from camera-area residents/buildings and emits bounded local work, construction completion, resource delivery, lamp ignition, wolf howl, and burial details. `StrategyAmbientAudioController` keeps the imported nature/weather/river layers and weights birds/insects/night by season, time, weather, map forest density, and wind.
- `StrategyDayNightCycleController` creates a runtime full-map sprite overlay sorted above world sprites and below preview/fog/UI, blends dawn/day/dusk/night tint colors over a 360-second scaled-time loop, exposes a `StrategyCalendarSnapshot` with display day, 24-hour clock, time-of-day phase, and derived season data, maps phases to Dawn 05:00-07:00, Morning 07:00-11:00, Noon 11:00-15:00, Afternoon 15:00-19:00, Dusk 19:00-22:00, and Night 22:00-05:00, reaches full visual darkness at the 22:00 Night boundary, sends dawn/nightfall/season-start messages to the event log, and blends the camera background color with the same phase without adding gameplay effects. `StrategySeasonCalendar` derives 7-day seasons in the explicit Spring, Summer, Autumn, Winter order and 28-day years from `DayIndex` while preserving the pre-existing enum numeric values.
- `StrategyFogOfWarController` renders runtime fog over the generated map with a 4-pixels-per-cell overlay, keeps persistent explored cells separate from current visibility, skips refresh work while player fog is disabled from F9, reduces camp/resident/building reveal radii during Dusk/Night/Dawn from the shared calendar phase, further reduces those radii during dense Fog weather, darkens explored-but-not-visible cells at night, replaces explored gray zones with light/medium/dense weather-fog bands during Fog weather while leaving visible cells clear, and keeps a daylight-range visibility mask for spawn systems that should ignore temporary sight loss. Gameplay visibility arrays still refresh globally, but texture repaint/upload is full-map only on first paint/fallback and chunk-local afterward.
- Fog visibility and texture refreshes are paced by unscaled real time so simulation speed does not multiply fog repaint work.
- `StrategyDebugPanelController` owns the F9 runtime debug overlay: it blocks world input while open, lets testing disable/enable player fog through `StrategyFogOfWarController.SetPlayerFogEnabled()`, toggles shared `StrategyDebugOptions` such as instant free construction, and forces weather states including Snow/Blizzard through `StrategyWeatherController.ForceWeatherSmooth()` so debug switching uses the normal gradual atmospheric transition.
- `StrategyPostProcessController` creates a runtime global URP Volume, enables post-processing on the strategy camera, and blends readable dawn/dusk/night color grading, bloom, and vignette from day/night phase plus weather rain/snow/cloud/fog/storm/wetness intensities and subtle seasonal color profiles.
- `StrategyCinematicVisualController` reuses or creates one runtime 2D global light, scans completed buildings including Bridges through the world chunk registry / active-building fallback, scans campfire emitters, scans roadside torch light sources through a roadside active registry, and configures reusable `StrategyCinematicLightEmitter` attachments. ## Current Hotspots

- Scene flow is name-gated through `StrategySceneCatalog`; additional gameplay/menu scenes must be registered there or moved to a stronger scene-role asset before a larger campaign/mode structure is added.
- Map generation/rendering remains a large texture pipeline. Cached worker painting reduced normal menu/direct map preparation from roughly 11 seconds to low-single-digit seconds, but it still allocates one roughly 36 MB pixel buffer before main-thread texture upload; keep one candidate only and monitor memory on lower-end targets.
- Input action names, IDs, UI bindings, and modal context ownership are now runtime contracts; change them together with `StrategyInputActionsContractTests` and router consumers.
- Build placement creates construction sites that finalize into runtime building records and participates in versioned save/load restoration. Confirmed demolition retires a building from new targets immediately, settles teardown at end of frame, captures all distinct physical stores plus pending production and exact household dish state, and leaves haulable piles before releasing the building. It does not yet provide zoning or long-term regional economy effects.
- House resources are local to each placed house and are included in settlement persistence. Household foraging from Houses is disabled; current normal ingredient intake is raw `Fish`/`Game`/`Eggs`/forage food delivery from Granaries, then the starter Caravan Cart while it has food, with a direct Hunter Camp/Fisher Hut/Forager Camp/Chicken Coop fallback when no stored food is available. Householders and children with displayed age 6+ can carry raw food to their own house, while Pottery delivery, Dish cooking, winter household Logs delivery, and recovery of demolished-house prepared dishes remain Householder-only. Existing crops can remain stored/consumed if present, but Garden Beds and crop production currently have no normal production path until crop production moves to external buildings. Recipes currently span Poor/Common/Hearty/Fine/Feast quality tiers, and quality affects ration value and HUD/debug context. Nightly household dinner consumes prepared recipe dishes first before falling back to direct ingredients; winter nights can consume stored house Logs for warmth. There is no global economy, storage cap, food spoilage, or direct morale system; saved building and loose-pile dish stacks restore by recipe ID while transient reservations are rebuilt.
- Storage yard Logs, Stone, Iron, Coal, Clay, Planks, Pottery, and Tools are uncapped and local to each placed storage yard but persist through the common store snapshot; storage logistics feeds construction reservations, production inputs, upgrades, household Pottery/winter Logs, and Granary hauling without creating a fungible global economy.
- Granary and starter-cart `Game`, `Fish`, `Eggs`, Berries, Roots, and Mushrooms are uncapped, local to their placed components, persisted through common stores, and aggregated by settlement resource queries; food logistics centralizes production-building raw ingredient stock through shared Haulers, and Householders can move stored stock into houses for cooking or reserve one unit directly from production-food buildings when no stored food is available. Food is not spoiled or included in trade offers yet.
- Weather and seasons affect visuals, ambience, wind, water animation, transition weights, forage pressure, outdoor temperature, house warmth, seasonal snow/ice cover, frozen-water fish/fishing blocking, winter readiness, and resident cold; the active weather and calendar time are persisted. Snow/Blizzard add visual snowfall and wind/cold post-process without rain audio or water ripples. Seasonal systems do not yet affect crops, fire risk, or storage spoilage.
- Generated Stone deposits are runtime-only map resource nodes mined by assigned stonecutters; starter-area deposits are guaranteed near the settlement spawn, but there is no Stone regrowth loop yet.
- POI-owned Iron fields are rebuilt runtime nodes for Mine worksites whose origin and remaining amount persist through their landmark; they stay walkable but block normal placement, can be mined underground by hidden Miners only under a Mine footprint, and feed Storage Yard Iron stock for Forge input.
- POI-owned Coal fields are rebuilt runtime nodes for Coal Pit worksites whose origin and remaining amount persist through their landmark; they stay walkable but block normal placement, can be mined by visible in-pit Coal Miners only under a Coal Pit footprint, and feed Storage Yard Coal stock for Kiln and Forge input.
- Generated Clay fields are runtime-only near-water map resource nodes for Clay Pit worksites; they stay walkable, block normal building placement, are inspectable, avoid adjacent Iron/Coal/Clay fields, can be reserved/mined by visible Clay Diggers only when under a Clay Pit footprint, and currently feed Storage Yard Clay stock for Kiln input without construction costs or a global economy.
- Wildlife is runtime-only. Deer and birds start ambient-only for the player, but adult deer can be reserved and hunted by upgraded Hunter Camps; rabbits can be reserved and hunted by hunter-camp workers for local `Game`, with missed arrows releasing and startling the target instead of creating a carcass; wolves are autonomous population-control predators that only hunt rabbit/deer surplus above high thresholds, chase the exact target position after reaching the target cell, and use a short fast pounce once close; current wildlife generation favors more numerous smaller groups over a few dense groups but no longer uses map-wide spawn, instead requiring daylight-range hidden near-settlement cells for initial spawn, reproduction birth cells, and migration targets; deer, rabbits, and wolves can swim across generated River cells only as transit toward a land target, do not treat Lake water as passable, and their land travel avoids placed buildings, active construction sites, and the campfire within a 4-cell structure buffer; lake fish can reproduce only inside hard-capped lake regions and per-shoal caps when a hidden near-settlement birth cell exists; river fish are non-reproductive pass-through agents spawned by a single timer from hidden near-settlement river-route cells; fish can be reserved and caught by fisher-hut workers from validated shore/land stand cells while the target remains in cast range and water is not frozen; and wildlife still does not block walkability, reveal fog, save, or feed a global economy.
- Residents currently move through the shared trail-aware navigation service around camp/home and job targets, start as 3 finite starter-camp family blocks with adult children, form and migrate households with kinship checks, consume nightly household dinners, accumulate nutrition/cold state, can die and trigger funerals, and work through capped worksites including manual Scout Lodges or uncapped settlement Hauler/Builder roles. Stable identity, age/life state, home/kinship links, nutrition, cold, and investigated point-of-interest state are persisted; worksite assignments, active movement, carried task reservations, Scout frontier/point reservations, funeral progress, and work-cycle internals are rebuilt after load. Residents do not yet have explicit marriage UI, maiden-name display, or broader economy effects.
- Auto workforce assignment is runtime-only and can rebalance overstaffed auto-managed professions by unassigning surplus workers through normal worksite APIs; if no free adult is available, higher-scored shortages can also pull limited donors from lower-priority auto-managed roles. Normal surplus and demand donor releases only take idle worksite workers so active movement/work cycles are not interrupted; explicit priority-0 profession shutdowns can still cancel active workers. Worksite reads use a cached snapshot rebuilt from world-chunk active building-component/site views on active-count changes plus a fallback real-time cadence, successful assignments are capped per tick, only residents who are immediately idle are reused in the same tick, and workers returning carried resources re-enter the candidate pool on later ticks. It intentionally does not force-reassign busy home duty or funeral tasks.
- Construction resource costs are currently runtime-local and paid through physical Logs/Stone/Planks in Storage Yards, loose construction piles, Lumberjack Camps, Stonecutter Camps, Sawmills, and the starter cart. Construction availability excludes existing construction reservations but not not-yet-picked logistics reservations; if construction claims that physical stock, matching logistics reservations are cleared, while already-carried resources stay excluded because pickup removes them from source stock. Production-building upgrade costs still spend Storage Yard stock directly. Carried construction resources dropped on resident death restore the original site's reservation when that site still needs the resource; late construction can require Planks, but no treasury, revenue, taxes, upkeep, or global stock service exists yet.
- Future risk will likely appear next in complete transient simulation-state persistence, runtime assembly cycles if subsystem asmdefs are split further, and broader economy state ownership.
