# Architecture Notes

Last updated: 2026-06-30

## Current Architecture

- The project is a Unity 2D/URP project with a thin runtime-generated MVP strategy layer.
- `StrategyGameBootstrap` runs after scene load, creates/wires the map, strategy wind, visual day/night cycle, runtime post-processing, cinematic visuals, ambience audio, forestry, population, nature props, settlement treasury, trade caravans, and camera without scene YAML dependencies, focuses the initial camera view on the startup campfire once population startup has created it, and configures nature after population so the camp clear radius can be excluded.
- `StrategyDebugLogger` is created before other strategy systems, writes structured session lines to project-root `debug.log` in the Unity Editor, mirrors Unity log messages, and exposes static `Info`/`Warn`/`Error` helpers for gameplay events.
- `StrategyRuntimeObjectCreationGuard` blocks loose-resource pile factories from creating new scene GameObjects while Play Mode is shutting down or when runtime code is not actually playing.
- `CityMapController` owns basic 192x192 default map-cell data, randomizes an active generation seed by default, derives a terrain-generation profile from that seed, tags water/shore cells by `CityMapWaterKind.River` or `CityMapWaterKind.Lake`, assigns visual-only `ReliefHeight` values, exposes `RiverFlowDirection` for current-following systems, renders the current map as a generated point-filtered texture on a `SpriteRenderer`, and logs `Map/Generated` total/subphase generation timings.
- `StrategyTerrainTexturePainter` paints the generated map texture with active-seed 16px pixel-art variants for Grass, Meadow, Forest, Dirt, Shore, and Water, including neighbor-aware side/corner transition overlays plus visual hill/mountain relief shading that does not affect gameplay walkability or buildability.
- `StrategyWaterAnimationController` renders one generated point-filtered overlay texture above the static map for animated water depth tint, directional river streaks, lake sparkles, broken shoreline foam, wet shore-edge pixels, and rain/storm ripple highlights, while repainting only cached water/shore cells after initial setup.
- `StrategyTrailController` now treats completed resident travel between two different non-Bridge buildings as the road source: after one real traversal, the route layer first tries a direct walkable line from the captured start cell to the final target cell, then falls back to smoothed resident route waypoints only when that direct line is blocked. Resident footfall accumulation no longer creates functional or visible roads, automatic route-network convergence is disabled, route roads do not decay from disuse, invalid route cells are pruned when map walkability or cell validity changes, and formed roads still provide a 15% resident speed multiplier plus reduced pathfinding cost. Resident road capture and any future road planning share `StrategyTrailPathfinder`, while `StrategyTrailRouteCellBuilder` bridges diagonal grid steps without completing small 2x2 square pockets and removes repeated-cell route loops before road cells are recorded. Each building pair keeps a still-valid canonical route so repeated traversals reinforce the same path instead of widening roads with nearby variants.
- `StrategyTrailSpriteFactory` generates connected procedural pixel-art trail sprites for cardinal N/E/S/W right-angle neighbor masks, 3 wear levels, and deterministic variants.
- `StrategyMapDistributionUtility` provides seeded shuffled full-map cell iteration and macro cluster scoring for capped map-object generation.
- `StrategyNaturePropController` is created after map generation and places trees, forest groups, and bushes over generated terrain using `CityMapController.ActiveSeed`, cell kinds, shuffled whole-map iteration, and vegetation cluster weighting; these props mark their occupied cells as not walkable.
- `StrategyNaturePropController` supports a cell-radius exclusion and currently skips generated nature props within 3 cells of the startup campfire.
- `StrategyNaturePropController` also places generated Stone deposits before vegetation on a cell: standalone Boulders, Rock Clusters, and larger Cliffs that register with `StrategyStoneResourceController`, block their occupied footprints, and use macro cluster weighting instead of purely per-cell random scatter.
- `StrategyNaturePropController` places generated underground Iron fields as multi-cell flat rust-stained ground and shallow vein markings on walkable land; they register with `StrategyIronResourceController`, remain walkable, block normal building placement through the map buildability overlay, avoid adjacent Coal fields, use macro cluster weighting, and can be mined by Mine worksites built over them.
- `StrategyNaturePropController` places generated underground Coal fields as multi-cell flat dark dust ground and coal seam markings on walkable land; they register with `StrategyCoalResourceController`, remain walkable, block normal building placement through the map buildability overlay, avoid adjacent Iron fields, use macro cluster weighting, and can be mined by Coal Pit worksites built over them.
- `StrategyNaturePropController` places generated near-water Clay fields as multi-cell wet clay patches/banks on walkable shore/land cells within a short water radius; they register with `StrategyClayResourceController`, remain walkable, block normal building placement, avoid adjacent Iron/Coal/Clay fields, use macro cluster weighting, and can be mined by Clay Pit worksites built over them.
- When the startup campfire exclusion center is known, `StrategyNaturePropController` guarantees at least 5 starter-area Stone deposits within `StrategyStonecutterCamp.WorkRadius`, plus 2 starter Coal and 2 starter Iron deposits in a nearby 10-24 cell ring before the general nature pass; Iron/Coal minimum fallbacks are not blocked by the decorative prop cap.
- `StrategyForageResourceController`, `StrategyForageNode`, and `StrategyForageSpriteFactory` create non-blocking Berries/Roots/Mushrooms forage nodes on suitable walkable land after nature generation through the shared shuffled whole-map pass plus forage cluster weighting, guarantee a small starter forage ring, support resident reservation, despawn gathered nodes immediately after interaction, and queue faster timed replacement spawns near mature standing trees inside active Forager Camp work radii. It keeps headroom above the initial generated node target, periodically supports under-supplied Forager Camps, and applies local cell-density plus per-camp soft caps so forage does not over-cluster in one area.
- `StrategyWildlifeController` runs after starter placement and fog/placement setup, builds lake fish regions and a river route from generated `CityMapWaterKind` cells, and spawns deer herds, rabbit groups, wolf packs, capped lake fish shoals, one-way pass-through river fish, and ambient birds only in currently hidden daylight-range fog cells within a broad ring around completed buildings or active construction sites; the startup camp is a fallback anchor only when no building/construction anchor exists. `StrategyWildlifeController.Part11.cs` owns the shared hidden-near-settlement candidate checks used by initial spawn, lake/river fish spawn, reproduction birth cells, and migration targets, while migration diagnostics keep a short cooldown for failed target cells so groups avoid repeated abort loops. Land wildlife travel avoids placed buildings, active construction sites, and the campfire within a 4-cell structure buffer while still allowing generated River crossings as transit-only path cells. Wolf pack placement prefers alternating river sides when a generated river route exists, wolf roam can recover from stale pack centers by falling back around the current wolf position and retargeting the pack center, and wolf prey lookup only reserves rabbit/deer surplus above high control thresholds after subtracting active predator reservations and hunter-reserved rabbits. `StrategyWildlifeRiverCrossing` separates land target cells from land-or-River travel cells: deer, rabbits, and wolves can path through generated River cells with slowed swimming movement and ripple visuals, but final path targets and hunter/wolf prey reservations remain on land cells without changing global map walkability or allowing Lake crossings. `StrategyDeerAgent` owns deer sex/life stage, fawn growth, predator reservation, state transitions, local pathing, resident/noisy-work threat checks, swimming crossings, threat-reaction throttling, and frame-based idle/walk/graze/alert/flee/rest animation; `StrategyRabbitAgent` owns the matching rabbit behavior set plus hunter reservation, predator reservation, arrow-hit, missed-arrow flee reaction, death, carcass, butchering-ready states, swimming crossings, and threat-reaction throttling; `StrategyWolfAgent` owns pack roaming, settlement-pressure avoidance, threshold-limited rabbit/deer hunting with a short fast pounce phase, swimming crossings, and frame-based idle/walk/run/stalk/attack/eat/howl animation; `StrategyFishAgent` owns lake-vs-river habitat state, water-cell pathing, river-route following/despawn, fry growth, swimming, feeding, turning, dart/flee behavior, fishing reservation, hooked/caught states, and surface ripple visuals; `StrategyBirdAgent` owns decorative bird state, habitat landing choices, flying/fleeing, pecking, hopping, duck swimming, and flight shadows.
- Wildlife service load is spread under time acceleration: migration categories are processed over successive frames, settlement cache and diagnostics cooldowns use real time, and rabbit/deer/bird/fish resident-threat scans use randomized real-time offsets while movement, aging, breeding, and production-facing outcomes remain scaled.
- `StrategyWildlifeController.Part13.cs` owns Hunter Camp-local support spawns: under-supplied camps periodically seed adult rabbits in a hidden 7-11 cell ring inside `StrategyHunterCamp.WorkRadius`, while upgraded camps can seed rare adult deer under stricter local caps. Support spawns reuse normal hidden wildlife candidate checks, structure buffers, global population caps, and local-density guards.
- `StrategyStoneResourceController` / `StrategyStoneDeposit` track runtime Stone resource nodes with deposit kind, footprint, remaining Stone amount, and reservation hooks for future worker jobs.
- `StrategyIronResourceController` / `StrategyIronDeposit` track runtime underground Iron indicators with deposit kind, footprint, remaining Iron amount, reservation state, and mining hooks for `StrategyMine`.
- `StrategyCoalResourceController` / `StrategyCoalDeposit` track runtime underground Coal indicators with deposit kind, footprint, remaining Coal amount, reservation state, and mining hooks for `StrategyCoalPit`.
- `StrategyClayResourceController` / `StrategyClayDeposit` track runtime near-water Clay fields with deposit kind, footprint, remaining Clay amount, reservation state, inspect data, buildability blocking, and mining hooks for `StrategyClayPit`.
- `StrategyForestryController` / `StrategyForestryTree` register generated standalone trees as mature forestry targets, block tree cells while trees exist, support hit-driven chopping, animate tree shake/cut marks/falling, keep fallen trunks as bucking targets, split them into pickup Logs, release walkability after Logs are taken, and plant saplings that grow through 3 visual stages. Small trees yield 3 Logs, while large generated trees and planted mature trees yield 6 Logs.
- `StrategyWindController` creates/configures a Unity directional `WindZone` as the runtime wind source, accepts weather influence boosts, and `StrategyWindSway` adapts the wind values into 2D sprite sway for generated trees, forest groups, and bushes.
- `StrategyWeatherController` owns the runtime weather state, randomly transitions between Clear, Cloudy, LightRain, HeavyRain, Fog, and Storm, exposes smooth rain/cloud/fog/storm/wind/wetness intensities, boosts the strategy wind source, feeds rain/wind ambience, and lets water animation intensify ripple hits during rain.
- `StrategyWeatherVisualController` renders procedural wet-ground, cloud-shadow, heavy-rain mist, and rain overlays in dedicated sorting bands around day/night, preview, and fog-of-war layers; dense Fog weather rendering is masked through fog-of-war instead of covering visible cells with a uniform mist overlay.
- `StrategyAmbientAudioController` loads non-generated nature loops from `Assets/Resources/Audio/Nature`, creates runtime ambience sources, follows map water cells with a spatial river source, and blends forest birds/cicadas/night/rain/wind based on time, map forest density, strategy wind values, and active weather.
- `StrategyDayNightCycleController` creates a runtime full-map sprite overlay sorted above world sprites and below preview/fog/UI, blends dawn/day/dusk/night tint colors over a 330-second scaled-time loop, exposes a `StrategyCalendarSnapshot` with display day, 24-hour clock, time-of-day phase, and phase accent color, maps phases to Dawn 05:00-07:00, Morning 07:00-11:00, Noon 11:00-15:00, Afternoon 15:00-19:00, Dusk 19:00-22:00, and Night 22:00-05:00, sends dawn/nightfall messages to the event log, and blends the camera background color with the same phase without adding gameplay effects.
- `StrategyFogOfWarController` renders runtime fog over the generated map with a 4-pixels-per-cell overlay, keeps persistent explored cells separate from current visibility, skips refresh work while player fog is disabled from F9, reduces camp/resident/building reveal radii during Dusk/Night/Dawn from the shared calendar phase, further reduces those radii during dense Fog weather, darkens explored-but-not-visible cells at night, replaces explored gray zones with light/medium/dense weather-fog bands during Fog weather while leaving visible cells clear, and keeps a daylight-range visibility mask for spawn systems that should ignore temporary sight loss.
- Fog visibility and texture refreshes are paced by unscaled real time so simulation speed does not multiply fog repaint work.
- `StrategyDebugPanelController` owns the F9 runtime debug overlay: it blocks world input while open, lets testing disable/enable player fog through `StrategyFogOfWarController.SetPlayerFogEnabled()`, toggles shared `StrategyDebugOptions` such as instant free construction, and forces weather states through `StrategyWeatherController.ForceWeatherSmooth()` so debug switching uses the normal gradual atmospheric transition.
- `StrategyPostProcessController` creates a runtime global URP Volume, enables post-processing on the strategy camera, and blends readable dawn/dusk/night color grading, bloom, and vignette from day/night phase plus weather rain/cloud/fog/storm/wetness intensities.
- `StrategyCinematicVisualController` reuses or creates one runtime 2D global light, scans completed buildings including Bridges plus the campfire for reusable `StrategyCinematicLightEmitter` attachments, renders a low-resolution camera-space nighttime darkness mask with soft light cutouts around active emitters, renders wet puddle glints and subtle foreground depth props around the active camera, and adds randomized rain/storm lightning flashes with occasional multi-flash bursts. It applies camera-distance LOD, caps active point lights, and updates darkness/puddles/foregrounds on short intervals rather than every frame. `StrategyCinematicLightEmitter` lazily creates local 2D point lights only when selected by the LOD budget, uses the campfire lit/ember/relight factor for campfire strength, while emissive pixel masks and animated torch/lantern/brazier/bridge-lamp sprites keep distant night light sources visible cheaply.
- Cinematic visuals, weather overlays, water animation, nature overlay frames, and light emitter visual ticks use unscaled real-time cadences where they are presentation work rather than gameplay progression.
- `StrategyShadowCaster2D` and `StrategyShadowSpriteFactory` provide shared procedural soft/cast ground shadows for runtime world sprites; day/night exposes global shadow opacity and length multipliers while each object type tunes its own shape, scale, offset, and sorting offset.
- `StrategyMusicController` loads all AudioClips from `Assets/Resources/Audio/Music` as an in-game playlist, picks random tracks, avoids repeating the previous track when 2+ tracks exist, and pauses/resumes the current clip on application focus loss without advancing the playlist.
- `StrategyResidentFootstepAudio` loads non-generated grass footsteps from `Assets/Resources/Audio/Footsteps/GrassWalk` and is attached by `StrategyResidentAgent` to play quiet spatial footstep one-shots on selected walk animation frames.
- `StrategyTimeScaleController` is runtime-created by bootstrap and maps F1/F2/F3 plus Build HUD x1/x2/x3 buttons to simulation speed by updating `Time.timeScale` and `Time.fixedDeltaTime`; modal gameplay systems use its pause-lock API so they can pause simulation without losing the requested speed. Core gameplay progression continues to use scaled `Time.deltaTime`, while performance-sensitive visual/service cadences use unscaled real time so x2/x3 does not multiply fog refreshes, cinematic visual scans/LOD, weather/water/nature overlay updates, trail service pruning, population household fallback checks, wildlife cache/log cooldowns, animal threat scans, wolf decision/path retry cadence, or resident path-build bursts.
- `StrategyNatureSpriteFactory` generates and caches procedural 2.5D pixel-art sprites for larger large trees, small trees, bushes, forest groups, Stone boulders/clusters/cliffs, walkable Iron-stained ground/vein indicators, walkable Coal dust/seam indicators, walkable wet Clay patch/bank indicators, felled trunks, split Logs, carried Logs, carried Iron, carried Coal, carried Clay, carried `Game`, and carried `Fish`.
- `StrategyResourceType`, `StrategyHouseResourceStore`, `StrategyHouseResourceStore.Dishes`, `StrategyDishRecipeCatalog`, and `StrategyResourceIconFactory` define the current resource layer, including Eggs/crops/forage food, recipe-based prepared dishes, `Logs`, `Stone`, `Iron`, `Coal`, `Clay`, `Planks`, `Pottery`, and `Tools`, plus shared runtime HUD icons for construction, production stock, and storage resources.
- `CityMapController` also owns counted dynamic walkability and buildability overlays used by placement/population for overlapping runtime blockers plus a bridge-walkability overlay that lets completed bridge spans make River water cells walkable without changing their water kind; these overlay changes notify the trail layer to refresh affected cells.
- `StrategyCameraController` owns orthographic navigation, initial focus requests, and clamps movement to the map bounds.
- `StrategyBuildMenuController` owns the first runtime HUD: a bottom Build button, category dock, optional subcategory dock for crowded categories, item tray, Logs/Stone/Planks construction costs, active build-tool state, resource affordability, temporary goal-driven tool locks, F9 instant-construction affordability bypass display, and full menu closing after placement; the catalog currently groups Extraction into Camps, Deposits, and Food subcategories while smaller categories stay flat.
- `StrategyProfessionHudController` owns the top-menu `Professions` HUD: it scans current worksite components, shows dynamic profession rows with generated icons and assigned/capacity counts, includes Miners for Mine worksites, Coal Miners for Coal Pits, Clay Diggers for Clay Pits, Potters for Kilns, and Blacksmiths for Forges, treats Storage Yard Haulers/builders as uncapped roles, exposes `Auto Assign` plus compact priority steppers, and routes player `-`/`+` assignment commands through each worksite's existing assignment API. Haulers are the single player-facing logistics profession and also fill Granaries.
- `StrategyAutoWorkforceController` is runtime-created by bootstrap and uses `StrategyAutoWorkforceSettings` plus demand records to assign eligible free adults every few real-time seconds. It keeps a throttled cached snapshot of current worksite arrays rebuilt from `StrategyPlacedBuilding.ActiveBuildings` and `StrategyConstructionSite.ActiveSites` on active-count changes plus a longer fallback interval instead of refreshing with scene-wide lookups every scaled simulation tick, skips repeated full demand/cache/donor scans for a longer cooldown after a no-free-adult tick produces no assignment or release, uses lightweight donor-failure diagnostics after failed donor searches, and logs `cacheRefreshMs` on tick diagnostics. It treats Construction, Food, Logistics, Wood, Stone, Planks, Iron, Coal, Clay, Pottery, and Tools values as desired worker targets for currently workable vacancies, while shortage, urgency, worksite capacity, storage backlog, production-input backlog, construction readiness, and resident distance affect demand ordering. Each tick also computes desired profession targets, releases surplus workers from overstaffed auto-managed roles, can release limited lower-priority auto-managed donors for higher-scored shortages through existing unassign APIs, avoids in-place sorting of cached worksite arrays when building capped demands, and caps successful assignments so large rebalances spread across later ticks. Manual worker removal from the Profession HUD creates a short profession override so automation does not immediately refill that role.
- `StrategyBuildPlacementController` owns MVP placement: map-cell hover preview, validation, construction-site creation, construction resource reservation, occupied cells, final placed runtime building records, expanded 2.5D walk-blocker footprints for houses and work camps, fisher-hut water/shore validation, Bridge's two-click river-bank span placement/highlighting, and tool-specific worksite component creation after completion, including the Trading Post trade endpoint and non-catalog starter Caravan Cart. Normal building construction-site validation keeps the technical foundation strict, rejects map buildability blockers such as underground Iron/Coal and near-water Clay, reserves the future final 2.5D blocker without requiring all of it to be currently walkable, requires nearby builder work access, and only blocks foundation cells while the site is under construction; Mine, Coal Pit, and Clay Pit can place over their matching resource cells. F9 instant construction mode skips resource reservation but still creates and immediately completes a construction site so building completion listeners and starter goals use the same finalization path; toggling it on also completes existing active construction sites.
- `StrategyBuildPlacementController` closes the full Build menu after successful construction-site placement and marks the placement frame so world selection does not auto-select the new site from the same click.
- Bootstrap uses `StrategyBuildPlacementController.TryPlaceStarterCaravanCart` to create a temporary starter Caravan Cart near the campfire with 20 Logs, 20 Stone, and randomized supported raw food covering 3 days for the initial population.
- `StrategyStarterCaravanCart` is a non-build-menu placed building using trade caravan art: it stores only initial stock, never accepts deliveries, exposes low-priority construction resource reservations after Storage Yards, transfers available Logs/Stone/Planks into completed Storage Yards, can dispatch a small temporary starter builder pool before a Storage Yard exists, lets Householders reserve raw food after Granaries and before production fallback, and despawns through placement demolition cleanup once all stock and reservations are gone.
- `StrategyPlacedBuilding` is the runtime record attached to placed buildings; it stores tool, origin, footprint, footprint bounds, selected visual variant, assigned resident references, installed visual upgrades, installed production-building upgrades, Bridge span cells/endpoints, exposes installed upgrade records, owns the house resource store, has a 2D click collider, and exposes `ActiveBuildings` plus `CopyActiveComponents<T>()` for performance-sensitive runtime queries that should not scan the whole scene.
- `StrategyConstructionSite` is the runtime bridge between placement and final buildings: it stores final tool metadata, target footprint, reserved/delivered Logs/Stone/Planks, an uncapped assigned builder list, staged construction sprites, delivered material stockpiles, optional Bridge span/work cells, progress, per-builder reserved build hits, a lightweight active-site registry, and completion into a final placed building; physically delivered materials define a buildable progress cap so builders can hammer partial work before every reserved resource arrives, and each builder must reserve one currently unlocked build hit before entering the hammer animation so parallel work cannot exceed delivered-material progress; it periodically requests hired Storage Yard builders or starter-cart temporary builders while waiting, and exposes a debug-only instant completion hook used by the F9 panel path.
- `StrategyStorageYard` is the current storage/logistics worksite: it stores uncapped local Logs, Stone, Iron, Coal, Clay, Planks, Pottery, and Tools, supports uncapped assigned Haulers plus uncapped hired builders, finds production worksites with available output stock, reserves Logs/Stone/Iron/Coal/Clay/Planks/Pottery/Tools for haulers, reserves non-food production inputs for delivery from Storage Yard stock into production nodes, exposes Pottery reservations for householder pickup, lets Haulers move `Game`/`Fish`/`Eggs`/forage food to Granaries, lets otherwise idle Haulers fallback-deliver reserved Logs/Stone/Planks to active construction sites, exposes non-food trade stock spend/receive helpers, fronts construction availability across loose piles, Storage Yards, production-local construction stock, and the low-priority starter cart, dispatches hired builders across active construction sites with balanced empty-site/low-builder and currently-buildable-site priority, and updates visible Logs/Stone/Iron/Coal/Clay/Planks/Pottery/Tools stockpiles.
- `IStrategyProductionLogisticsNode` is the shared production-chain logistics contract: production buildings expose input delivery requests and output pickup requests, while Storage Yard Haulers perform inter-building resource transfers through Storage Yard stock. Food-chain inputs should use Granary-side logistics when they are introduced.
- `StrategyProductionStorage` centralizes the current production-building local stock cap of 6 resources; production worksites use it to stop starting new work when a local buffer would overflow, while Storage Yards and Granaries stay uncapped.
- `StrategyProductionBuildingUpgradeCatalog` defines one Tools-based upgrade per current extraction/production building; Storage Yards pay upgrade costs from available `Tools`, `Planks`, and `Stone`, while installed upgrades live on the placed building and are read by resident work cycles.
- `StrategySettlementTreasury`, `StrategyTradeCaravanController`, `StrategyTradeTransactionService`, and `StrategyTradingPost` form the current trade MVP: a runtime Coin treasury, a cached/periodic caravan visitor, fixed buy/sell offers, and transaction helpers that sell available Storage Yard/Granary stock or buy goods back into the nearest valid storage building.
- `StrategyLumberjackCamp` remains the lumber production worksite, but its capped local Logs stock can now be reserved and picked up by Haulers.
- `StrategySawmill` is the Planks production worksite and first `IStrategyProductionLogisticsNode`: it supports 1 assigned Sawyer, requests Logs as a Storage Yard delivered production input, keeps the Sawyer visible inside the building with a saw/log/plank overlay plus sawdust effects, stores capped local Logs/Planks stock with pending Planks counting against the same cap, caps input Logs at 4 to preserve output room under the shared production cap, and exposes Planks to Storage Yard haulers as output.
- `StrategyKiln` is the Pottery production worksite and an `IStrategyProductionLogisticsNode`: it supports 1 assigned Potter, requests Clay and Coal as Storage Yard delivered production inputs, keeps the Potter visible at the Kiln with a firing overlay plus spark/dust effects, stores capped local Clay/Coal/Pottery stock with pending Pottery counting against the same cap, and exposes Pottery to Storage Yard haulers as output.
- `StrategyForge` is the Tools production worksite and an `IStrategyProductionLogisticsNode`: it supports 1 assigned Blacksmith, requests Iron, Coal, and Logs as Storage Yard delivered production inputs, keeps the Blacksmith visible at the Forge with a forging overlay plus spark effects, stores capped local Iron/Coal/Logs/Tools stock with pending Tools counting against the same cap, consumes `1 Iron + 1 Coal + 1 Log` into `1 Tools`, and exposes Tools to Storage Yard haulers as output.
- `StrategyMine` is the Iron production worksite: it requires an available underground Iron indicator under its footprint, supports up to 2 assigned Miners, keeps Miners hidden underground during active work while the entrance emits intermittent dust/spark effects, stores capped local Iron stock, and exposes that stock to Storage Yard haulers.
- `StrategyCoalPit` is the Coal production worksite: it requires an available underground Coal indicator under its footprint, supports up to 2 assigned Coal Miners, keeps workers visible inside the pit during active work with coal dust/chip effects, stores capped local Coal stock, and exposes that stock to Storage Yard haulers.
- `StrategyClayPit` is the Clay production worksite: it requires an available near-water Clay field under its footprint, supports up to 2 assigned Clay Diggers, keeps workers visible inside the pit during active work, stores capped local Clay stock, and exposes that stock to Storage Yard haulers.
- `StrategyHunterCamp` is the first hunting worksite: it supports up to 2 assigned hunters, reserves the nearest available adult rabbit on the map through `StrategyWildlifeController`, benefits from camp-local wildlife support spawns when nearby huntable animals are below target, unlocks adult deer hunting through the Deer Hunting Kit production upgrade, uses resident-side ranged bow stand selection with a 20% miss chance, receives capped local `Game`, and updates a local visual stockpile.
- `StrategyFisherHut` is the first fishing worksite: it supports up to 2 assigned fishers, reserves the nearest available adult fish on the map through `StrategyWildlifeController`, chooses reachable land/shore stand cells with adjacent water, keeps range validation active through cast/wait/reel phases, receives capped local `Fish`, and updates a local visual stockpile.
- `StrategyForagerCamp` is the first external forage-food worksite: it supports up to 2 assigned Foragers, reserves generated Berries/Roots/Mushrooms nodes, stores capped local forage food stock, and exposes that stock to Granary Haulers.
- `StrategyChickenCoop` is a standalone external egg-production building with no worker slots: it uses a `4x4` placement footprint, a larger standalone wrapper around the legacy coop sprite frames, the shared production stock cap, autonomous `Eggs` production, slightly larger idle chickens around the placed building, visual night shelter/release, Granary Hauler and Householder fallback pickup, and stock/timer context in selection HUD.
- `StrategyGranary` is the first food-storage worksite: it stores uncapped local raw `Game`, `Fish`, `Eggs`, Berries, Roots, and Mushrooms ingredients, exposes settlement-level availability/reservation helpers for Householder pickup and trade, is filled by Storage Yard Haulers, and updates separate visual food stockpiles.
- `StrategyBuildingUpgradeController` still contains the legacy Garden Beds and House Chicken Coop implementation, but normal House flow no longer auto-installs Garden Beds and the selected-house HUD no longer exposes Garden Beds or Chicken Coop actions. Treat this code as inactive legacy scaffolding until crop production moves to standalone external buildings or the old subsystem is removed.
- `StrategyBuildingUpgradeSpriteFactory` generates and caches runtime pixel-art sprites for the current visual upgrades.
- `StrategyPopulationController` creates a startup camp with an animated procedural campfire on a walkable land cell at least 6 cells from generated water/shore when possible, spawns 3 initial family blocks with a father, a mother, and 1-2 adult children each, assigns random Germanic/Nordic-style full names and age-appropriate adult ages, owns the runtime resident ID, family-record, house registry, orphan-adoption pass, and unsettled accepted-refugee family tracker, populates completed houses from whole homeless families before falling back to the homeless adult pair pool and adult-child migration/partner lookup, and funnels resident death through assignment cleanup plus funeral snapshot creation; workplace and construction roles do not prevent home assignment, accepted refugee families that lack housing are not split by generic pair assignment before all members share one house, and residents in active funeral duty are temporarily immune to death attempts.
- `StrategyRefugeeArrivalController` gates the first pending refugee family on the scripted evening of Day 2, schedules later families by random repeat interval, adds one daily dynamic workforce-demand roll after that first scripted family, fades arrival intensity after 40 accepted residents, stops arrivals at 50 accepted residents, stages them inside the map about 4 cells beyond a random side of the daylight-visible fog boundary, validates walkable routes to the campfire before spawning, pauses on arrival through `StrategyTimeScaleController`, and either registers accepted families as normal residents or sends rejected temporary agents back to the hidden arrival staging point for cleanup. The dynamic roll compares accepted adult count against finite worker slots from capped worksites plus one construction slot per active construction site, and intentionally ignores uncapped Storage Yard Hauler/builder roles.
- `StrategyTopStatusHudController` is a small runtime-created top HUD layer that reads adult/child counts from `StrategyPopulationController` and calendar/time snapshots from `StrategyDayNightCycleController`; pending refugees are not counted until accepted, the compact population panel toggles `StrategyPopulationRosterHudController`, and a separate top-right widget shows day number, 24-hour clock time, phase label, phase accent, and day progress. The roster HUD lists accepted residents with settlement stats, filters, age, home/camp state, role, current status, and food status, using `StrategyResidentHudText` for shared labels with the selection HUD. The roster also opens `StrategyFamilyTreeHudController`, a fullscreen modal family-tree scene that pushes a `StrategyTimeScaleController` pause lock, uses permanent horizontal/vertical scrollbars, groups recorded members into connected same-surname family cards, lays those cards out as affinity-ordered left-to-right family columns with compact generation rows, and draws parent-child portrait-card connections plus cross-family relationship lines for living and deceased records. Deceased tree cards are muted monochrome cards with a skull marker, every member card shows a gender symbol, and hovering a card shows relationship labels on other recorded relatives. `StrategyEventLogHudController` is a separate compact top event feed for births, deaths, adoptions, dawn, and nightfall. `StrategyGoalsController` and `StrategyGoalsHudController` provide the runtime checklist layer; `StrategyStarterGoalSequenceController` seeds the first onboarding build goals and uses Build menu tool locks until the player completes 3 Houses, a Forager Camp, then a Lumberjack Camp and Stonecutter Camp.
- `StrategyPopulationController` periodically checks empty and single-resident houses: empty houses receive the oldest adult child still living with parents, and single adult-child households try to pull in an adult opposite-gender partner from another parental home or the free camp pool while using kinship checks. When an adult male/female household pair forms, the wife takes the husband's family name and the resident family record is refreshed without rewriting biological parent/child IDs.
- Population empty-house/adult-child migration fallback checks run on unscaled real time, while event-driven house-completion and death paths remain immediate.
- `StrategyHouseholdState` lives on occupied house buildings and runs the randomized household birth timer; it asks `StrategyPopulationController` to spawn children only for adult male/female pairs that pass `StrategyKinshipUtility` close-relative checks and when the house has a free resident slot.
- `StrategyHouseholdFoodState` lives on occupied house buildings and resolves one nightly dinner per day after a one-day settling grace. Dinner waits for eligible residents to enter the house during `Night`, falls back after a short deadline if someone is stuck, sums resident age-based ration needs, consumes prepared house-local recipe dishes first, falls back to house-local ingredients for missing rations, applies short rations to each resident's nutrition debt/days hungry, and exposes aggregate food status for HUDs while keeping consumed dish/ingredient counts for context.

## Current Hotspots

- Runtime bootstrap is global for every loaded scene. Future menus/multi-scene flow may need an explicit boot scene or mode gate.
- Map generation/rendering is texture-based and lightweight for the MVP, now using a randomized seed profile for terrain kinds and visual relief height, a procedural terrain painter for visual variants/transitions/relief shading, a runtime trail overlay, and separate runtime nature props that can block cells. Future per-cell interaction may still need Tilemap, mesh picking, or a dedicated map model/service.
- Camera and Build menu controls use direct Input System device reads. Future configurable controls should move to input actions.
- Build placement is runtime-only and creates construction sites that finalize into runtime building records. It does not yet create durable city state, save data, zoning, or long-term economy effects.
- House resources are runtime-only and local to each placed house. Household foraging from Houses is disabled; current normal ingredient intake is raw `Fish`/`Game`/`Eggs`/forage food delivery from Granaries, then the starter Caravan Cart while it has food, with a direct Hunter Camp/Fisher Hut/Forager Camp/Chicken Coop fallback when no stored food is available. Householders and children with displayed age 6+ can carry raw food to their own house, while Pottery delivery from Storage Yards and Dish cooking remain Householder-only. Existing crops can remain stored/consumed if present, but Garden Beds and crop production currently have no normal production path until crop production moves to external buildings. Recipes currently span Poor/Common/Hearty/Fine/Feast quality tiers, and quality affects ration value and HUD/debug context. Nightly household dinner consumes prepared recipe dishes first before falling back to direct ingredients. There is no global economy, save data, storage cap, trade, spoilage, morale effect, or saved recipe state yet.
- Storage yard Logs, Stone, Iron, Coal, Clay, Planks, Pottery, and Tools are runtime-only, uncapped, and local to each placed storage yard; storage logistics feeds local construction reservations, production-input deliveries, production-building upgrade payments, householder Pottery pickup reservations, and food hauling into Granaries, but this still does not create a global economy or save data.
- Granary and starter-cart `Game`, `Fish`, `Eggs`, Berries, Roots, and Mushrooms are runtime-only, uncapped, and local to their placed building components; food logistics centralizes production-building raw ingredient stock through shared Haulers, and Householders can move stored stock into houses for cooking or reserve one unit directly from production-food buildings when no stored food is available. Food is not spoiled, saved, traded, or exposed through a settlement-level stock service yet.
- Weather is runtime-only and currently affects visuals, ambience, wind, and water animation; it does not yet affect crops, movement, visibility, illness, fire risk, or save data.
- Generated Stone deposits are runtime-only map resource nodes mined by assigned stonecutters; starter-area deposits are guaranteed near the settlement spawn, but there is no Stone regrowth loop yet.
- Generated Iron fields are runtime-only map resource nodes for Mine worksites; they stay walkable but block normal building placement, are inspectable, avoid adjacent Coal fields, can be reserved/mined underground by hidden Miners only when under a Mine footprint, and currently feed Storage Yard Iron stock for Forge input without construction costs or a global economy.
- Generated Coal fields are runtime-only map resource nodes for Coal Pit worksites; they stay walkable but block normal building placement, are inspectable, avoid adjacent Iron fields, can be reserved/mined by visible in-pit Coal Miners only when under a Coal Pit footprint, and currently feed Storage Yard Coal stock for Kiln and Forge input without construction costs or a global economy.
- Generated Clay fields are runtime-only near-water map resource nodes for Clay Pit worksites; they stay walkable, block normal building placement, are inspectable, avoid adjacent Iron/Coal/Clay fields, can be reserved/mined by visible Clay Diggers only when under a Clay Pit footprint, and currently feed Storage Yard Clay stock for Kiln input without construction costs or a global economy.
- Wildlife is runtime-only. Deer and birds start ambient-only for the player, but adult deer can be reserved and hunted by upgraded Hunter Camps; rabbits can be reserved and hunted by hunter-camp workers for local `Game`, with missed arrows releasing and startling the target instead of creating a carcass; wolves are autonomous population-control predators that only hunt rabbit/deer surplus above high thresholds, chase the exact target position after reaching the target cell, and use a short fast pounce once close; current wildlife generation favors more numerous smaller groups over a few dense groups but no longer uses map-wide spawn, instead requiring daylight-range hidden near-settlement cells for initial spawn, reproduction birth cells, and migration targets; deer, rabbits, and wolves can swim across generated River cells only as transit toward a land target, do not treat Lake water as passable, and their land travel avoids placed buildings, active construction sites, and the campfire within a 4-cell structure buffer; lake fish can reproduce only inside hard-capped lake regions and per-shoal caps when a hidden near-settlement birth cell exists; river fish are non-reproductive pass-through agents spawned by a single timer from hidden near-settlement river-route cells; fish can be reserved and caught by fisher-hut workers from validated shore/land stand cells while the target remains in cast range; and wildlife still does not block walkability, reveal fog, save, or feed a global economy.
- Population is runtime-only. Residents currently move over trail-aware A* grid paths around camp/home and job targets, start as 3 finite starter-camp family blocks with adult children, move into houses after house construction based on home/family/pair state rather than job state, can migrate from parental homes into empty houses as adult children, can form simple opposite-gender household pairs with kinship checks where the wife takes the husband's family name, can have children inside occupied houses, consume nightly household dinners from prepared house `Dish` with per-resident malnutrition state, no longer forage directly from Houses, can fetch raw Granary food or direct production-food fallback as Householders or displayed-age-6+ children helping their own home, plus Storage Yard Pottery and cooked ingredients as Householders only, can die from annual age-based mortality with malnutrition multipliers and trigger runtime funerals/graves unless currently in funeral duty, can be manually assigned to work camps/storage yards as adults, can hunt rabbits through Hunter Camps, fish through Fisher Huts, forage through Forager Camps, and can be hired as Storage Yard builders for construction; assigned jobs remain assigned at night but new work cycles are paused outside Dawn-through-Dusk settlement work time, housed idle residents sleep hidden inside homes during `Night` while notifying household dinner readiness, and homeless idle residents visibly sleep around the startup campfire after relighting the cold or embered campfire if needed. Residents do not have save data, explicit marriage UI, maiden-name display, or broader economy effects.
- Auto workforce assignment is runtime-only and can rebalance overstaffed auto-managed professions by unassigning surplus workers through normal worksite APIs; if no free adult is available, higher-scored shortages can also pull limited donors from lower-priority auto-managed roles. Normal surplus and demand donor releases only take idle worksite workers so active movement/work cycles are not interrupted; explicit priority-0 profession shutdowns can still cancel active workers. Worksite reads use a cached snapshot rebuilt from active placed buildings and active construction sites on active-count changes plus a fallback real-time cadence, successful assignments are capped per tick, only residents who are immediately idle are reused in the same tick, and workers returning carried resources re-enter the candidate pool on later ticks. It intentionally does not force-reassign busy home duty or funeral tasks.

Production-building upgrade costs still spend Storage Yard stock directly. Carried construction resources dropped on resident death restore the original site's reservation when that site still needs the resource; late construction can require Planks, but no treasury, revenue, taxes, upkeep, or global stock service exists yet.
