# Architecture Notes

Last updated: 2026-06-12

## Current Architecture

- The project is a Unity 2D/URP project with a thin runtime-generated MVP strategy layer.
- `StrategyGameBootstrap` runs after scene load, creates/wires the map, strategy wind, visual day/night cycle, ambience audio, forestry, population, nature props, and camera without scene YAML dependencies, focuses the initial camera view on the startup campfire once population startup has created it, and configures nature after population so the camp clear radius can be excluded.
- `StrategyDebugLogger` is created before other strategy systems, writes structured session lines to project-root `debug.log` in the Unity Editor, mirrors Unity log messages, and exposes static `Info`/`Warn`/`Error` helpers for gameplay events.
- `CityMapController` owns basic map-cell data, randomizes an active generation seed by default, derives a terrain-generation profile from that seed, tags water/shore cells by `CityMapWaterKind.River` or `CityMapWaterKind.Lake`, exposes `RiverFlowDirection` for current-following systems, and renders the current map as a generated point-filtered texture on a `SpriteRenderer`.
- `StrategyTerrainTexturePainter` paints the generated map texture with active-seed 16px pixel-art variants for Grass, Meadow, Forest, Dirt, Shore, and Water, including neighbor-aware side/corner transition overlays.
- `StrategyNaturePropController` is created after map generation and places trees, forest groups, and bushes over generated terrain using `CityMapController.ActiveSeed` and cell kinds; these props mark their occupied cells as not walkable.
- `StrategyNaturePropController` supports a cell-radius exclusion and currently skips generated nature props within 3 cells of the startup campfire.
- `StrategyNaturePropController` also places generated Stone deposits before vegetation on a cell: standalone Boulders, Rock Clusters, and larger Cliffs that register with `StrategyStoneResourceController` and block their occupied footprints.
- When the startup campfire exclusion center is known, `StrategyNaturePropController` guarantees at least 5 starter-area Stone deposits within `StrategyStonecutterCamp.WorkRadius`, outside the camp clear radius and with adjacent walkable work cells.
- `StrategyWildlifeController` runs after starter placement, finds suitable walkable land away from the campfire for deer/rabbits, builds lake fish regions and a river route from generated `CityMapWaterKind` cells, finds species-appropriate habitat cells for decorative birds, spawns small deer herds, rabbit groups, capped lake fish populations, one-way pass-through river fish, and ambient birds, manages capped reproduction for deer/rabbits/lake fish, exposes adult-rabbit reservation/count lookup for hunter camps, and exposes catchable-fish reservation/count lookup for fisher huts; `StrategyDeerAgent` owns deer sex/life stage, fawn growth, state transitions, local pathing, resident/noisy-work threat checks, and frame-based idle/walk/graze/alert/flee/rest animation; `StrategyRabbitAgent` owns the matching rabbit behavior set plus hunter reservation, arrow-hit, death, carcass, and butchering-ready states; `StrategyFishAgent` owns lake-vs-river habitat state, water-cell pathing, river-route following/despawn, fry growth, swimming, feeding, turning, dart/flee behavior, fishing reservation, hooked/caught states, and surface ripple visuals; `StrategyBirdAgent` owns decorative bird state, habitat landing choices, flying/fleeing, pecking, hopping, duck swimming, and flight shadows.
- `StrategyStoneResourceController` / `StrategyStoneDeposit` track runtime Stone resource nodes with deposit kind, footprint, remaining Stone amount, and reservation hooks for future worker jobs.
- `StrategyForestryController` / `StrategyForestryTree` register generated standalone trees as mature forestry targets, block tree cells while trees exist, support hit-driven chopping, animate tree shake/cut marks/falling, keep fallen trunks as bucking targets, split them into pickup Logs, release walkability after Logs are taken, and plant saplings that grow through 3 visual stages.
- `StrategyWindController` creates/configures a Unity directional `WindZone` as the runtime wind source; `StrategyWindSway` adapts the wind values into 2D sprite sway for generated trees, forest groups, and bushes.
- `StrategyAmbientAudioController` loads non-generated nature loops from `Assets/Resources/Audio/Nature`, creates runtime ambience sources, follows map water cells with a spatial river source, and blends forest birds/cicadas/night/rain/wind based on time, map forest density, and strategy wind values.
- `StrategyDayNightCycleController` creates a runtime full-map sprite overlay sorted above world sprites and below preview/fog/UI, blends dawn/day/dusk/night tint colors over a 220-second scaled-time loop, and blends the camera background color with the same phase without adding gameplay effects.
- `StrategyMusicController` loads all AudioClips from `Assets/Resources/Audio/Music` as an in-game playlist, picks random tracks, avoids repeating the previous track when 2+ tracks exist, and pauses/resumes the current clip on application focus loss without advancing the playlist.
- `StrategyResidentFootstepAudio` loads non-generated grass footsteps from `Assets/Resources/Audio/Footsteps/GrassWalk` and is attached by `StrategyResidentAgent` to play quiet spatial footstep one-shots on selected walk animation frames.
- `StrategyTimeScaleController` is runtime-created by bootstrap and maps F1/F2/F3 plus Build HUD x1/x2/x3 buttons to simulation speed by updating `Time.timeScale` and `Time.fixedDeltaTime`; modal gameplay systems use its pause-lock API so they can pause simulation without losing the requested speed.
- `StrategyNatureSpriteFactory` generates and caches procedural 2.5D pixel-art sprites for large trees, small trees, bushes, forest groups, Stone boulders/clusters/cliffs, felled trunks, split Logs, carried Logs, carried `Game`, and carried `Fish`.
- `StrategyResourceType`, `StrategyHouseResourceStore`, and `StrategyResourceIconFactory` define the current house-local resource layer plus shared runtime HUD icons for construction, hunter-camp, and fisher-hut stock resources.
- `CityMapController` also owns a counted dynamic walkability layer used by placement/population for overlapping runtime blockers plus a bridge-walkability overlay that lets completed bridge spans make River water cells walkable without changing their water kind.
- `StrategyCameraController` owns orthographic navigation, initial focus requests, and clamps movement to the map bounds.
- `StrategyBuildMenuController` owns the first runtime HUD: a bottom Build button, category dock, item tray, Logs/Stone construction costs, active build-tool state, resource affordability, and full menu closing after placement; the catalog includes Infrastructure/Bridge.
- `StrategyProfessionHudController` owns the top-menu `Professions` HUD: it scans current worksite components, shows dynamic profession rows with generated icons and assigned/capacity counts, treats Storage Yard storage workers/builders as uncapped roles, and routes player `-`/`+` assignment commands through each worksite's existing assignment API.
- `StrategyBuildPlacementController` owns MVP placement: map-cell hover preview, validation, construction-site creation, construction resource reservation, occupied cells, final placed runtime building records, expanded 2.5D walk-blocker footprints for houses and work camps, fisher-hut water/shore validation, Bridge's two-click river-bank span placement/highlighting, and tool-specific worksite component creation after completion. Normal building construction-site validation keeps the technical foundation strict, reserves the future final 2.5D blocker without requiring all of it to be currently walkable, requires nearby builder work access, and only blocks foundation cells while the site is under construction; construction sites can be accepted while waiting for hired Storage Yard builders.
- `StrategyBuildPlacementController` closes the full Build menu after successful construction-site placement and marks the placement frame so world selection does not auto-select the new site from the same click.
- Bootstrap uses `StrategyBuildPlacementController.TryPlaceStarterStorageYard` to create the initial Storage Yard near the campfire through the same placed-building path as player construction.
- `StrategyPlacedBuilding` is the runtime record attached to placed buildings; it stores tool, origin, footprint, footprint bounds, selected visual variant, assigned resident references, installed visual upgrades, Bridge span cells/endpoints, exposes installed upgrade records, owns the house resource store, and has a 2D click collider.
- `StrategyConstructionSite` is the runtime bridge between placement and final buildings: it stores final tool metadata, target footprint, reserved/delivered resources, up to 2 hired builders, staged construction sprites, delivered material stockpiles, optional Bridge span/work cells, progress, and completion into a final placed building; it periodically requests builders from Storage Yards while waiting.
- `StrategyStorageYard` is the current storage/logistics worksite: it stores local Logs and Stone, supports uncapped assigned logistics workers plus uncapped hired builders, finds production camps with available camp stock, reserves Logs/Stone for haulers and construction sites, provides construction pickup cells, dispatches hired builders to waiting sites, and updates visible Logs/Stone stockpiles.
- `StrategyLumberjackCamp` remains the lumber production worksite, but its local Logs stock can now be reserved and picked up by storage workers.
- `StrategyHunterCamp` is the first hunting worksite: it supports up to 2 assigned hunters, reserves nearby adult rabbits through `StrategyWildlifeController`, receives delivered `Game`, and updates a local visual stockpile.
- `StrategyFisherHut` is the first fishing worksite: it supports up to 2 assigned fishers, reserves nearby adult fish through `StrategyWildlifeController`, chooses reachable shore cells for casting, receives delivered `Fish`, and updates a local visual stockpile.
- `StrategyGranary` is the first food-storage/logistics worksite: it stores local `Game` and `Fish`, supports up to 2 assigned food haulers, reserves available food from Hunter Camps/Fisher Huts, exposes settlement-level food availability/consumption helpers for households, and updates separate visual food stockpiles.
- `StrategyBuildingUpgradeController` installs house upgrades near placed houses. It currently supports Garden Beds and Chicken Coop, charges small Logs/Stone costs from available Storage Yard resources, assigns produced resources to upgrades, uses map walkability only as a placement filter, does not block cells after installation, and spawns idle chickens when Chicken Coop is installed.
- `StrategyBuildingUpgradeSpriteFactory` generates and caches runtime pixel-art sprites for the current visual upgrades.
- `StrategyPopulationController` creates a startup camp with an animated procedural campfire, spawns 3 male and 3 female initial residents, assigns random Germanic/Nordic-style full names and ages 18-30, owns the runtime resident ID, family-record, and house registries, populates completed houses from the homeless adult pair pool before falling back to adult-child migration/partner lookup, and funnels resident death through assignment cleanup plus funeral snapshot creation; workplace and construction roles do not prevent home assignment.
- `StrategyRefugeeArrivalController` gates the first pending refugee family on 3 completed houses, schedules later families by random repeat interval, routes them from a walkable map edge to the campfire, pauses on arrival through `StrategyTimeScaleController`, and either registers accepted families as normal residents or sends rejected temporary agents back off-map for cleanup.
- `StrategyTopStatusHudController` is a small runtime-created top HUD layer that reads adult/child counts from `StrategyPopulationController`; pending refugees are not counted until accepted.
- `StrategyPopulationController` periodically checks empty and single-resident houses: empty houses receive the oldest adult child still living with parents, and single adult-child households try to pull in an adult opposite-gender partner from another parental home or the free camp pool while using kinship checks.
- `StrategyHouseholdState` lives on occupied house buildings and runs the randomized household birth timer; it asks `StrategyPopulationController` to spawn children only for adult male/female pairs that pass `StrategyKinshipUtility` close-relative checks and when the house has a free resident slot.
- `StrategyHouseholdFoodState` lives on occupied house buildings and periodically consumes `Game`/`Fish` from Granaries based on resident count; unmet food demand raises a starvation level that multiplies residents' annual mortality chance through `StrategyPopulationController`.
- `StrategyResidentAgent` stores a persistent runtime full name, family name, age, life stage, parent IDs, and child ID links, drives simple idle movement around the current camp/home through short walkable grid paths using `CityMapController.IsCellWalkable`, owns visual-only readability child renderers for a synced silhouette outline plus ground shadow, and attaches `StrategyResidentFootstepAudio` for quiet grass footsteps on walk frames; children younger than 3 stay inside their assigned home with hidden world sprite/collider and skip outdoor idle/funeral movement, older children idle/walk near home, cannot be assigned to work/build, grow into adults at age 16 after scaled game time, continue aging after adulthood, and roll annual mortality through `StrategyPopulationController` from age 1. Resident funeral activities can temporarily interrupt work for mourning, procession, and burial while keeping workplace roles intact afterward. Assigned adult residents also periodically path to their home Garden Beds, play a short work animation, and add the Garden Beds crop to the owning house when work completes. Residents assigned to a lumberjack camp path to nearby trees, play frame-based axe swings, send tree hit events on impact frames, wait for falling to finish, buck the fallen trunk into Logs, carry Logs to their camp stockpile, then plant saplings nearby. Residents assigned to a hunter camp reserve nearby adult rabbits, move into bow range, fire projectile arrows, approach carcasses, butcher them, carry `Game`, and deposit it at the camp. Residents assigned to a fisher hut reserve fish, move to reachable shore, cast and reel fishing lines, carry `Fish`, and deposit it at the hut. Residents assigned to a granary reserve stored `Game`/`Fish` from production buildings, carry it to the granary, and deposit it into food stock. Residents hired as Storage Yard builders fetch reserved Logs/Stone from Storage Yards, deliver materials to construction sites, then build with frame-based hammer swings and timed progress hits.
- `StrategyFuneralController`, `StrategyCorpse`, `StrategyCemeteryController`, and `StrategyFuneralSpriteFactory` own the current funeral/cemetery MVP: deaths create animated corpse snapshots, family/household participants gather and cry/mourn, the corpse is dragged by rope behind a carrier toward a spontaneous cemetery, burial waits for reachable attendees at the grave, burial creates a runtime grave sprite, and the grave cell becomes not walkable.
- Home assignment on `StrategyResidentAgent` updates the resident's family/home binding without cancelling active workplace or construction tasks; idle residents can immediately walk toward the new home, while busy residents keep their current task flow and later idle around the assigned home.
- Storage workers are also driven by `StrategyResidentAgent`: they reserve camp Logs/Stone, walk to the source camp, pick up resources, carry them to the storage yard, and deposit them.
- `StrategyCampfireSpriteFactory` and `StrategyCampfireAnimator` generate and cycle the current campfire animation frames at runtime; `StrategyCampfireAnimator` also blocks the campfire cell while burning, fades/shrinks the fire during burnout, destroys the visual object, and clears that map walkability blocker when extinguished.
- `StrategyChickenAgent` drives small idle chickens around their linked Chicken Coop using the same early local walkability approach.
- `StrategyDeerSpriteFactory` generates procedural 2.5D male/female deer sprite frames, including larger antlered bucks and smaller does, for idle breathing, walking, grazing, alert, running, and resting behaviors.
- `StrategyRabbitSpriteFactory` generates procedural 2.5D male/female rabbit sprite frames for idle, hopping, nibbling, alert, fleeing, grooming, resting, arrow hit, death, and carcass behaviors; kits reuse scaled rabbit visuals.
- `StrategyFishSpriteFactory` generates procedural fish sprite frames for Minnow, Carp, and Perch idle, swimming, dart/flee, turn, feed, and hooked/reeling behaviors; fry reuse scaled fish visuals.
- `StrategyBirdSpriteFactory` generates procedural 2.5D bird sprite frames for Sparrow, Crow, and Duck idle, pecking, hopping, flying, landing, and duck swimming behaviors.
- `StrategyBuildingSpriteFactory` generates and caches runtime building sprites in code; House has 5 larger 2.5D variants, Lumberjack Camp has 3 variants, Stonecutter Camp has 3 variants, Hunter Camp has 3 variants, Fisher Hut has 3 variants, Storage Yard has 3 variants, Granary has 3 variants, and Bridge has dynamic span-sized sprites. Variant 0 is used by menu/preview and random variants are used for placed supported buildings.
- `StrategyResidentSpriteFactory` generates and caches 5 male and 5 female adult resident sprites, child resident sprites, walk frames, woodcut axe-swing frames, stonecut pickaxe frames, bow-shot frames, butchering frames, fishing cast/reel frames, construction hammer/build frames, and matching portrait sprites at runtime.
- `StrategyBuildingSpriteFactory` now also generates Storage Yard Logs/Stone stockpile sprites, Hunter Camp `Game` stockpile sprites, Fisher Hut `Fish` stockpile sprites, and Granary `Game`/`Fish` stockpile sprites.
- `StrategyConstructionSpriteFactory` generates staged construction-site sprites and delivered construction Logs/Stone stockpile sprites.
- `StrategyWoodcutEffectAnimator` generates short procedural woodchip/leaf/dust hit effects for tree chopping.
- `StrategyChickenSpriteFactory` generates and caches the current runtime chicken sprite.
- `StrategyWorldSelectionController` handles left-click world selection for placed buildings/construction sites/residents, ignores the placement-completion frame, displays a runtime selection marker, owns the right-side selection HUD panel, shows selected-object previews/status/context, shows selected-house resident portrait rows, exposes compact house visual-upgrade action rows, and shows selected-house resource icons/counts.
- `StrategyWorldSelectionController` also shows worksite stock/source context and construction-site cost/delivery/builder/progress context, but player-facing worker assignment/removal now lives in `StrategyProfessionHudController`.
- Rendering is configured through URP assets under `Assets/` and `Assets/Settings/`.
- `StrategyWorldSorting` centralizes 2.5D draw order: terrain/water/fog/preview use fixed bands, while world sprites use a world-Y anchor so lower/nearer objects render in front of higher/farther ones.
- Input is available through Unity's Input System. MVP camera, Build menu, fog toggle, and time-scale controls read direct keyboard/mouse devices instead of the default action asset.
- Scene content is currently the default sample scene/template setup.

## Current Hotspots

- Runtime bootstrap is global for every loaded scene. Future menus/multi-scene flow may need an explicit boot scene or mode gate.
- Map generation/rendering is texture-based and lightweight for the MVP, now using a randomized seed profile for terrain kinds, a procedural terrain painter for visual variants/transitions, and separate runtime nature props that can block cells. Future per-cell interaction may still need Tilemap, mesh picking, or a dedicated map model/service.
- Camera and Build menu controls use direct Input System device reads. Future configurable controls should move to input actions.
- Build placement is runtime-only and creates construction sites that finalize into runtime building records. It does not yet create durable city state, save data, zoning, or long-term economy effects.
- House resources are runtime-only and local to each placed house. Garden Beds currently cost 2 Logs/1 Stone and produce one assigned crop through resident work; Chicken Coop costs 4 Logs/2 Stone and passively produces Eggs. There is no global economy, save data, storage cap, trade, or consumption yet.
- Storage yard Logs and Stone are runtime-only and local to each placed storage yard; storage logistics now feeds local construction reservations, but does not yet feed a global economy, save data, or consumption loop.
- Granary `Game` and `Fish` are runtime-only and local to each placed granary; food logistics centralizes production-building stock, but food is not consumed, spoiled, saved, traded, or exposed through a settlement-level stock service yet.
- Generated Stone deposits are runtime-only map resource nodes mined by assigned stonecutters; starter-area deposits are guaranteed near the settlement spawn, but there is no Stone regrowth loop yet.
- Wildlife is runtime-only. Deer and birds are still ambient-only; rabbits can be reserved and hunted by hunter-camp workers for local `Game`; lake fish can reproduce only inside hard-capped lake regions; river fish are non-reproductive pass-through agents spawned by a single timer along `RiverFlowDirection`; fish can be reserved and caught by fisher-hut workers for local `Fish`; and wildlife still does not block walkability, reveal fog, save, or feed a global economy.
- Population is runtime-only. Residents currently idle locally around camp/home with short grid paths, move into houses from a finite starter-camp pool after house construction while that pool exists based on home/pair state rather than job state, can migrate from parental homes into empty houses as adult children, can form simple opposite-gender household pairs with kinship checks, can have children inside occupied houses, consume household food from Granaries, can die from annual age-based mortality with starvation multipliers and trigger runtime funerals/graves, can be manually assigned to work camps/storage yards as adults, can hunt rabbits through Hunter Camps, fish through Fisher Huts, and can be hired as Storage Yard builders for construction, but do not have save data, marriage UI, or broader economy effects.
- Construction resource costs are currently runtime-local and paid through Storage Yard Logs/Stone reservations; no treasury, revenue, taxes, upkeep, or global stock service exists yet.
- Future risk will likely appear next in scene bootstrapping, input routing, economy state ownership, and map persistence.

## Refactor Notes

- Keep map generation, map simulation, and map rendering separable once economy/zoning starts.
- Keep Build menu catalog/selection separate from placement rules. Add tools/buildings gradually by explicit user request.
- Move occupancy and placed-building records out of the placement controller when persistence or simulation starts.
- Move bootstrap from global runtime initialization to an explicit scene/root object if the project gains menus, save loading, or multiple game modes.
- Avoid larger abstractions until the first economy/zoning loop exists.
